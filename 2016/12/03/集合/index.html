集合

Iterable

所有Iterable的子类，
都能使用迭代器Iterator来完成所存储元素的迭代输出

Collection

List

1、元素存入顺序和取出顺序一致（有序）
2、元素可以重复
3、可以使用get(i)的方式将元素取出
4、List中删除、查找、判断是否包含依据是equals方法。

ArrayList

1、底层原理：Object[]
2、输出方式：

  • 可以使用for结合get(i)方式输出
  • 可以使用Iterator迭代输出
  • 可以使用ListIterator方式迭代输出

3、常用方法

Vector

和ArrayList区别：
1、ArrayList/Vector的区别，可以参考StringBuilder/StringBuffer、HashMap/Hashtable

  • 推出时间不一致
  • 单线程执行（线程不安全）/支持多线程（线程安全）
  • 效率较高/效率较低

2、可以使用Enumeration接口迭代输出。
3、当存储元素存满时，扩展长度不一样，ArrayList是扩展1/2，Vector是扩展两倍。

LinkedList

1、底层:

  • 采用双向链表

2、使用场景：

  • 对于需要频繁插入和删除数据的场景，可以使用本类。

3、和ArrayList的区别：

  • ArrayList底层是数组，LinkedList底层是双向链表，底层原理的不同决定了使用场景的不同。
  • LinkedList中提供了一系列链表操作的方法。

Set

1、无序（存入顺序和取出顺序不一致），注意无序不是随机顺序。
2、自动去重，不可以重复
3、只能使用迭代器输出，不能像List一样使用for循环结合get(i)的方式

HashSet

1、元素散列排放，无序
2、元素不能重复，元素的去重利用的是hashCode和equals方法

TreeSet

1、存入的元素要求可比较
2、会根据比较规则，自动将元素排序
如何让元素具有可比较性？
1）、实体类实现Comparable接口，重写compareTo方法
2）、TreeSet构造中传入比较器Comparator的子类，子类中重写compare方法，让TreeSet自身具有比较元素的能力。

Map

1、数据以键值对的方式存储。
2、键不能重复、无序（Set的特点）。
3、跟Iterable没有直接联系，不能直接使用迭代器取得全部元素
4、可以使用values方法得到所有的value
5、可以使用keySet方式，得到所有key的set集合。
6、可以使用entrySet方法得到Map.Entry的set集合，一个Map.Entry就包含了一个key和一个value

HashMap

1、HashSet的底层是HashMap，所以HashMap的key的特点和HashSet的特点一模一样

Hashtable

跟HashMap的区别：
1、不支持null
2、和StringBuilder/StringBuffer、ArrayList/Vector一样。

TreeMap

1、TreeSet的底层是TreeMap，所以TreeSet的key的特点和TreeSet一模一样。
2、一般来讲，都是String和八种基本数据类型的包装类做key

迭代器

Iterator

1、针对所有Iterable的子类，对Map无效。

ListIterator

1、专门针对List提供的迭代器
2、提供了add和remove方法，避免了并发修改异常。
3、提供了逆序迭代的能力

foreach循环

1、内部原理是Iterator，也需要注意并发修改问题

Enumeration

1、用于操作Vector
2、此接口的功能与 Iterator 接口的功能是重复的。此外，Iterator 接口添加了一个可选的移除操作，并使用较短的方法名。新的实现应该优先考虑使用 Iterator 接口而不是 Enumeration 接口。

工具类

Arrays

1、asList：将数组转为一个List集合。
2、binarySearch：二分查找数组中某一元素的下标
3、copyOf：数组拷贝
4、copyOfRange：数组范围拷贝
5、sort：排序
6、toString：将数组格式化输出

Collections

1、copy：完成List的拷贝
2、max/min：得到集合元素的最大值/最小值
3、reverse：将Lis集合的元素反转
4、sort：将List排序，可以传入比较器。
5、swap：交换List中两个下标所对应的元素
